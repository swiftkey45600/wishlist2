# Wishlist Service

Архитектурная заготовка сервиса вишлистов подарков.

## Описание

Это skeleton проекта без реализации бизнес-логики, базы данных и API. Содержит только структуру классов и методов-заглушек.

## Сущности

### User
Пользователь системы с логином, паролем и профилем.

### Event
Событие (день рождения, свадьба и т.д.), к которому привязаны подарки. Имеет публичную ссылку для друзей.

### Gift
Подарок в рамках события. Содержит название, описание, цену, ссылку на маркетплейс.

### Contribution
Взнос на подарок. Позволяет нескольким людям скидываться на дорогой подарок.

### Reservation
Бронирование подарка одним человеком (альтернатива совместному сбору).

## Killer Feature

**Совместный сбор на подарок** — несколько друзей могут скинуться на дорогой подарок:
- `price` — полная стоимость подарка
- `collected` — уже собрано денег
- `remaining` — осталось собрать
- `progress_percent` — процент выполнения

Логика расчета находится в `GiftService.get_gift_progress()`.

## Как делить работу

1. **Backend-разработчик**: реализовать repository-слой с реальной БД (PostgreSQL/SQLite)
2. **Backend-разработчик**: реализовать service-слой с бизнес-логикой
3. **Backend-разработчик**: добавить FastAPI/Flask для REST API
4. **Frontend-разработчик**: создать UI для создания событий и добавления подарков
5. **Frontend-разработчик**: создать публичную страницу события для друзей
6. **DevOps**: настроить деплой, CI/CD, мониторинг
